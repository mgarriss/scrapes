            <h2 class="entryTitle">The Soda Languages
  <span class="entryPermalink"><a href="http://redhanded.hobix.com/cult/theSodaLanguages.html"
    title="Permanent link to &#8220;The Soda Languages&#8221;">#</a></span>

by whytheluckystiff.net/“ 

  title="Visit why's homepage">why</a>
  in <a href="/cult" title="List of entries in the &#8220;cult&#8221; category">cult</a>


             <div class="entryContent">

            <p>Whoa, get ready to spend your whole afternoon tooling around with a small pile of little languages and compilers.  I am completely hooked on <a href="http://piumarta.com/pepsi/">Idst</a>, a compiler for a hybrid prototype-based Smalltalk-plus-C language by Ian Piumarta. And atop the bedrock of Idst: the soda languages.</p>

        <p>I first heard about Ian&#8217;s stuff in a #camping discussion where <a href="http://piumarta.com/pepsi/pepsi.html">Pepsi</a> (the Smalltalk/C hybrid) was brought up.  Well, it turns out Idst also comes with a Lisp-like soda language called Jolt  (see <strong>examples/jolt</strong>.)  Other examples are just great and include a teensy Squeak VM (<strong>examples/sqvm</strong>) and some <span class="caps">X11</span> samples (<strong>examples/x11</strong>.)</p>

        <p>He explains the Pepsi syntax in <a href="http://weeklysqueak.wordpress.com/2006/10/16/dynamic-messages-a-tour-of-pepsi/">an interview</a> a few days ago on the Weekly Squeak:</p>

        <blockquote>
                <p>You can mix C-code and “Id” code in a very simple way. So it is easy to integrate with O.S. services. The idea is quite the opposite of Objective-C: you think in terms of objects all the time (as in SmallTalk).  Then you “come back” to C-Language for the dirty part of your work.</p>
        </blockquote>

        <p>The syntax is suprisingly terse.  You use curly braces to break out into a method written in C.  Square brackets for Smalltalk-style.</p>




 X11Display newWindow  [ ^X11Window withDisplay: self ]

 X11Display _createSimpleWindow_: _x _: _y _: _w _: _h _: _bw _: _b _: _bg
 {
   return (oop)XCreateSimpleWindow((Display *)self-&gt;v__dpy,
     DefaultRootWindow((Display *)self-&gt;v__dpy),
     (int)v__x, (int)v__y, (int)v__w, (int)v__h,
     (int)v__bw, (int)v__b, (int)v__bg);
 }

 X11Display createWindow: bounds borderWidth: borderWidth
   border: border background: background
 [
   ^self newWindow
  _win_: (self _createSimpleWindow_: (bounds _x) _: (bounds _y)
   _: (bounds _w) _: (bounds _h)
   _: borderWidth _integerValue
   _: border _integerValue
   _: background _integerValue)
 ]






        <p>Can you get over that?</p>

        <p class="update"><strong>Update:</strong> Some of the above interview appears to be taken from <a href="http://article.gmane.org/gmane.comp.lang.smalltalk.squeak.general/107455">a post</a> on the Smalltalk newsgroup, which goes into greater detail concerning Coke and Jolt, as well as the other literal man-decades of work Ian has assigned himself.</p>

        <blockquote>
                <p>It&#8217;s often said that Smalltalk and Lisp suffer because they are systems that eat all of their young.  If <span class="caps">COLA</span> succeeds it might be called the system that ate its parents.</p>
        </blockquote>

    title="Permanent link to &#8220;5.gets BradPauly&#8221;">#</a></span>

by whytheluckystiff.net/“ 

  title="Visit why's homepage">why</a>
  in <a href="/5.gets" title="List of entries in the &#8220;5.gets&#8221; category">5.gets</a>


             <div class="entryContent">

            <p class="intro">This <a href="http://www.foundmagazine.com/find/719">frog skin</a> found in a used book is served up by Mongrel!  I can&#8217;t think of a better payload.  And the minute I heard on the Mongrel mailing list that <a href="http://foundmagazine.com/">Found Magazine</a> [<a href="http://en.wikipedia.org/wiki/Found_Magazine">W</a>] had been coded afresh in Ruby, I hit up Brad Pauly, the developer, for some discussion.</p>

        <p class="gets">1. So, Found must be a fun site to work on because you&#8217;ve just got                                                              this pile                                                                                                                          of litter (notes, photos, street junk and so on) and you&#8217;re                                                                        basically interfacing                                                                                                              technology with that mound of other folk&#8217;s discarded trash.  How do                                                                you sort it                                                                                                                        all out?  Or is it best to just leave it disorganized?</p>

        <p class="puts">Yeah, I had a ton of fun working on it. The previous version had some organization by way of &#8220;related finds&#8221; that we left out on purpose                                                                  for this version. People seem to prefer the randomness and it makes                                                                 sense to me. I think the joy of discovery is diminished when you&#8217;re                                                                 guided.</p>

        <p class="gets">(cont&#8217;d) ...Do you think all the                                                                   trouble                                                                                                                            we spend sorting things ever pays off?  Sometimes it seems like                                                                    when things are                                                                                                                    sorted, everyone ends up seeing the same things, in the same order.</p>

        <p class="puts">To the extent that categorizing/tagging makes information more                                                                    accessible I think it&#8217;s useful. And that&#8217;s about it. I don&#8217;t think it                                                               influences the experience much. I&#8217;m wondering how things might look                                                                 without categories.</p>

        <p class="gets">2. It seems like the whole site is artistic in nature.  Like: this                                                                 trash is                                                                                                                           beautiful when out of context and when it&#8217;s all warped and                                                                         damaged.  Are there                                                                                                                any libraries or algorithms that feel artful to you?  That go well                                                                 with art?</p>

        <p class="puts">Definitely, but in a different way. I think beauty in code comes from                                                               mastery more than context. High levels of skill with a tool allow                                                                   creativity to flow. So anything that encourages creation&#8230;</p>

        <p class="puts">I&#8217;ve been programming for a while but my

tool belt still has that stiff leather feel. I can&8217;t decide what goes where and I keep swapping the hammer for a pair of pliers, using that for a while, then switching back.

        <p class="gets">3. You&#8217;re using Mongrel on the site.  Wow.  Has it been a cinch?</p>

        <p class="puts">It&#8217;s been great. The server is a dedicated box that is managed with                                                                 Plesk and also hosts a couple other sites. I hadn&#8217;t used Plesk before                                                               and didn&#8217;t want to mess up the other sites or end up with a config                                                                  that was overly complex. I was able to add one config file that was                                                                 no more than ten lines long (a few rewrite rules and ProxyPass                                                                      stuff), restart Apache, and the site was live &#8211; serving the other                                                                   sites, the <span class="caps">FOUND</span> blog (WordPress), and handing off to Mongrel for the                                                               main site. It&#8217;s been humming along for a couple months now without                                                                  any problems.</p>

        <p class="gets">4. Hey, give us a freaky pic from the Found catalogue.  Anything                                                                   that just                                                                                                                          totally took you guys off guard or just made you die for hours.</p>

        <p class="puts">Hehe. So many, and I&#8217;ve only seen the ones that get published. This                                                               one gets a lot of play.. still my favorite though: <a href="http://www.foundmagazine.com/find/558">#558</a></p>

        <p class="gets">5. Lastly, and this is just out of curiousity, how do you use <span class="caps">RSS</span>?                                                                 Do you use a                                                                                                                       reader regularly?  Is it a part of your life or anything like that?</p>

        <p class="puts">Nothing fancy. I recently used it to add feeds to a simple forum I                                                                built for Rails Day. NetNewsWire is the reader I use and it typically                                                               sits next to Mail. When I check email I usually give <span class="caps">NNN</span> a refresh too.</p>

            <p>There&#8217;s an article that covers <a href="http://shanesbrain.net/articles/2006/10/02/screen-scraping-wikipedia">taking apart Wikipedia pages</a> with Hpricot on the blog of Shane Vitarana.  His commenters also point out the <a href="http://en.wikipedia.org/w/query.php">Wikipedia <span class="caps">API</span></a>, which is a fantastic way of getting at the raw data&#8212;in <span class="caps">YAML</span> even!         <p>Anyway, I just want to make a couple suggestions for his script.  These aren&#8217;t a big deal, just some shortcuts in Hpricot, which aren&#8217;t documented yet.  Let&#8217;s start getting them known, you know?</p>

        <p>First, a simple one: the Element#attributes hash can be used through Element#[].</p>




 #change /wiki/ links to point to full wikipedia path
 (content/:a).each do |link|
   unless link['href'].nil?
     if link['href'] =~ %r!^/wiki/!
       link['href'].sub!('/wiki/', 'http://en.wikipedia.org/wiki/')
     end
   end
 end






        <p>In the section where Shane loops through a bunch of <span class="caps">CSS</span> selector and removes everything that matches, I think it&#8217;s quicker to join the separate <span class="caps">CSS</span> selectors with commas, to form a large selector which Hpricot can find with a single pass.</p>




 #remove unnecessary content and edit links
 (content/items_to_remove.join(',')).remove






        <p>And lastly, I&#8217;ve <a href="http://code.whytheluckystiff.net/hpricot/changeset/54">checked in</a> a new <code>swap</code> method in which replaces an element with an <span class="caps">HTML</span> fragment.</p>




 #replace links to create new entries with plain text
 (content/"a.new").each do |link|
   link.swap(link['title'])
 end






        <p>So, yeah.  Little things.  Hey, are there any other methods you&#8217;re itching for?</p>

        <p>Hey, whoa, Aaron Patterson&#8217;s got a great hack!  File.open("out.jpg", 'wb') { |a|
   a &lt;&lt; obj.write_simple(";D thx aaron!")
 }

  <span class="entryPermalink"><a href="http://redhanded.hobix.com/inspect/theNewStringIsnTEnumerable.html"

  in <a href="/inspect" title="List of entries in the &#8220;inspect&#8221; category">inspect</a>


             <div class="entryContent">

            <p>So, it sounds like the 1.9 series will remove <code>Enumerable</code> from <code>String</code>&#8217;s ancestry, in favor of <code>String#lines</code>.  All string iteration casts to an Array. Maybe I&#8217;m getting my hopes up, but all these string changes seem like portents of Unicode Times.</p>




 &gt;&gt; str = "pony\nwagon\nstungun\n"
 =&gt; "pony\nwagon\nstungun\n"
 &gt;&gt; str.max
 NoMethodError: undefined method `max' for "pony\nwagon\nstungun\n":String
        from (irb):2:in `Kernel#binding'
 &gt;&gt; str.lines.max
 =&gt; "wagon\n"






        <p>Makes sense.  Anyone else mourning the impotent string splat?</p>




 &gt;&gt; utilities = *str
 =&gt; ["pony\nwagon\nstungun\n"]






        <p>Like a little rotary saw whose tooth got stuck.  (from <a href="http://www.rubyist.net/~matz/20060921.html#p03">matz</a> and <a href="http://www.ruby-lang.org/cgi-bin/cvsweb.cgi/ruby/string.c?cvsroot=src&#38;r1=1.274&#38;r2=1.275">patch</a>.)</p>

            <h2 class="entryTitle">Open Up Your Wiki and We'll Finish it For You
  <span class="entryPermalink"><a href="http://redhanded.hobix.com/inspect/howToLetAnyoneElseFinishYourWiki.html"
    title="Permanent link to &#8220;Open Up Your Wiki and We'll Finish it For You&#8221;">#</a></span>

by whytheluckystiff.net/“ 

  title="Visit why's homepage">why</a>
  in <a href="/inspect" title="List of entries in the &#8220;inspect&#8221; category">inspect</a>


             <div class="entryContent">

            <p>I&#8217;m hoping the Freaky Freaky Sandbox will avail me of having to finish anything ever again.  Instead, I can just stop.  Whatever it is, leave it unfinished.  Gently place a sandboxed wiki on it.  And then all of you at home can each add five lines of code and that&#8217;s it!</p>

        <p style="text-align:center;"><img src="/images/tippytippytepee-1.png" alt="" /></p>

        <p>So, let&#8217;s get started on this sandboxed scriptable wiki.  Step one, get some embedded Ruby in there.  Here&#8217;s how.</p>




 %w(sandbox rubygems redcloth
    camping acts_as_versioned).each { |lib| require lib }

 Camping.goes :Tepee

 Tepee::Box = Sandbox.safe
 Tepee::Box.load "erbl.rb"






        <p>I&#8217;m using <a href="http://code.whytheluckystiff.net/svn/camping/trunk/examples/tepee.rb">Tepee</a> as the base.  Sandbox gets loaded first because if you load RubyGems first, it goes bonkers.  So, we&#8217;re going to start up a little barebones sandbox and load the ERbLight library, which is written in pure Ruby.  Just to keep it simple.</p>

        <p>Now to swap out the markup processor for a sandbox eval.</p>




  def markup str
    @boxx = nil
    begin
      str = Tepee::Box.eval("ERbLight.new(#{str.dump}).result")
    rescue Sandbox::Exception =&gt; @boxx
    end
    RedCloth.new(str, [ :hard_breaks ]).to_html
  end






        <p>And that&#8217;s it.  If the sandbox throws an exception, it&#8217;ll go in the <code>@boxx</code> ivar.  It might be better to check for an exception when the user saves the page instead.</p>

        <p>So, this sandbox does <em>nothing</em>.  I mean you can do math and string stuff and hashes and whatever, but it&#8217;s not that great.  But come back in a few days and I&#8217;ll show you some more tricks for the <a href="http://code.whytheluckystiff.net/svn/sandbox/trunk/examples/tippytippytepee/">Tippy Tippy Tepee</a>.</p>
